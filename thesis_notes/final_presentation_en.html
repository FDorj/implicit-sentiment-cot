Implicit Sentiment Analysis1
Problem

Why implicit sentiment is harder

No explicit sentiment word

The polarity often comes from context, contrast, or common-sense implications.

Target-specific judgment

The same sentence may contain multiple targets; the label must be assigned to one target only.

Reasoning matters

The model must connect target, aspect, implied opinion, and polarity.

Mini example

Sentence: "Got this Mac Mini with OS X Mountain Lion for my wife." Target: "OS X Mountain Lion". The sentence mentions the target, but it does not imply sentiment toward it, so the correct label is neutral.

Project question

Can multi-step reasoning become more useful for ISA when it is paired with error diagnosis, controller logic, and source selection?

Accuracy

Meaning: the percentage of test examples with the correct label. Why used: it gives a simple overall correctness score.

Macro-F1

F1 is the harmonic mean of precision and recall: F1 = 2 * P * R / (P + R). Macro-F1 computes F1 for each class, then averages them equally.

Controller

Why a controller was necessary

Reflection can over-correct

A diagnostic label should not automatically replace all other predictions.

Confidence matters

Low-confidence diagnostics should be treated differently from high-confidence ones.

Agreement is strong evidence

If Direct and THOR agree, the controller can keep the shared label.

Implemented logic

The controller reads direct label, THOR label, diagnostic label, error type, and confidence, then applies a conservative rule order.

1. Normalize

Normalize labels, error type, and confidence before any decision.

2. Keep agreement

If Direct and THOR agree, keep that shared label.

3. Accept correction

If the error type is correctable and confidence meets the threshold, use the diagnostic label.

4. Fall back

Otherwise use the fallback source, usually Direct; then THOR or diagnostic if needed.

Source Selection

Why train-calibrated selection was added

Observation

No single source was best in every situation. Some profiles favored Direct, some favored THOR, and some diagnostic signals were useful as decision context.

Policy idea

Use the train split to learn which source is most reliable for each profile, then apply that decision table to test without using test labels.

Reproducibility17
Scope

What was intentionally not claimed

Why not fine-tune Qwen3 8B here

Ollama is inference-oriented. Fine-tuning Qwen3 8B would require a separate LoRA/QLoRA setup, suitable GPU resources, careful validation, and more training data than the 1,746 ISA train examples provide.

Flan-T5 not evaluated

A HuggingFace runner scaffold exists, but Flan-T5 results are not part of the validated experiments.

No API model comparison

OpenAI API or other external API models were not included in current result files.

THOR-style implementation

The project uses a compatible THOR-style pipeline, not an exact reproduction of the official codebase.

Fine-tuning status

Qwen3 8B weights were not updated. The current contribution is pipeline design: prompting, diagnostic reflection, controller logic, and train-calibrated source selection.
